# Terms used across the xPack Project

### xPack

**xPack** stands for **xpm package**. It is a npm package with an additional
`xpack` definitions to store the **xpm** metadata.

It is also used as a **generic name** for the top project (**The xPack Reproducible Build Framework**, or, in short **The xPack Project**).

### xpm

**xpm** stands for the **xPack Project Manager**. It is a Node.js CLI
application. It is NOT a replacement for **npm**, but it complements its
functionality with features specific to
compiled languages like C/C++.

### npm

Although the existing documentation does not acknowledge the acronym, **npm**
is the Node.js package manager.

### Node.js

**Node.js** is a cross-platform, open-source JavaScript runtime
environment for CLI applications.

### CLI

**CLI** stands for **Command-Line Interface**. When used with
program names, it indicates that the program runs in a computer terminal,
as opposed to a graphical user interface.

### npmjs.com

**[npmjs.com](https://npmjs.com)** is the public repository of Node.js
packages. Access is free and there are millions of packages, with
tens of billions of downloads every week.

### CDL

**CDL** stands for **Component Definition Language**. It was introduced
by eCos and is documented in [The eCos Component Writer's Guide](http://ecos.sourceware.org/docs-3.0/cdl-guide/cdl-guide.html).

### xCDL

**xCDL** is the xPack Configuration Definition Language and framework,
inspired by CDL.

### Cross-platform tool

A tool that can be used across multiple different platforms (like Windows,
macOS, GNU/Linux).

### Reproducibility

For software development,
[reproducibility](https://en.wikipedia.org/wiki/Reproducibility)
is the characteristic of a
development frameowrk that provides stability over time and platforms,
such that the same builds or tests can be execute again and again with
identical results.

### User global xPacks store

A platform-dependent folder within the home folder where **xpm** installs
all versions of the packages.
